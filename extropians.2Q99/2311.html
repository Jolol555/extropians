The newsfilter, at least at first, will run on the server, that means the user doesn't have to download and install anything. Some of the functions eventually can be performed on the "smart" client, starting with profile and advisor list caching.

The first implementation will be Usenet, will be easily extendable to any other message stream as long as it has a way of addressing messages uniquely. Static information repositories such as Web :-) are much easier to handle, except for the size.

We'll have to start from simple features as the team is very small, but can do more faster if we get help :-)

The language of the project will be Java.

More details - on the project page and in the mailing list archive.
