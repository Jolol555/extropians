On Sat, 27 Feb 1999, Brian Atkins wrote:

> Why not combine collective and collaborative filtering then?
> First implement some collective filtering to get rid of off
> topic and kookish posts. Then later on add onto it another
> layer of software allowing for collective filtering for each
> individual's interests?

Example SIG: 

1) Kookify 	[http://some.url.com/mail.cgi?kookify=memberId]
2) Glorify 	[http://some.url.com/mail.cgi?glorify=memberId]
3) Kill Thread	[http://some.url.com/mail.cgi?killThread=threadSubject]
4) Kill Topic	[http://some.url.com/mail.cgi?killTopic=topicType
-----------------------------------------------------------------------

Explanation:

  1. Clicking Kookify will vote to have the poster of the message labelled a kook. The user will no longer receive messages posted by the kook. If a certain percentage of the list members kookify the person, they will be filtered globally.
  2. Glorifying a poster will have the effect of casting a vote to keep the person from ever being banned.
  3. Kill Thread does the obvious. You will no longer recieve that thread. (and possibly not receive dereived threads either?)
  4. Kill Topic. If subjects are prefixed correctly (ie. ADV, META, SCI/TECH...) you can elect to have certain specific topics filtered.

I would gladly write the software to do this if I can be compensated for my time.
