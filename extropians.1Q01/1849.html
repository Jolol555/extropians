Barbara Lamar wrote:
> This was a page that was converted from a Word document to HTML .

That's your problem right there. ^_-

IMO, the best way to convert non-HTML documents to HTML - for all the
converters people have tried over the years - remains "by hand". It
doesn't take very many minutes to learn the basics of raw HTML, and the
end result is almost always worth the extra time to convert (if for no
other reason than it doesn't wind up looking like crap). In fact, I
usually edit non-plain-text documents in raw HTML if I get to choose
the format (usually only when I'm creating the doc in the first place),
using a text editor (not a WYSIWIG HTML editor), since I can usually
get acceptable results faster, and better results in the same time.
