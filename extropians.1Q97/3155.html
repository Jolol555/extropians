Been there, saw that. This system covers backlinking, but it doesn't
answer the problem of permanent storage. Whoever owns the server
determines how long the server will keep data before pitching it.
Worse, this system doesn't appear to be easily scalable to the worldwide
database that we all would hope that hypertext would allow.

My Usenet idea could do everything this could, and it would be easier to
implement on a wide scale... Just implementing that comparably
miniscule change in the "news:" URL would be enough to bring hypertext
to every Usenet reader in the world. Yet both ideas neglect the issue
of a permanent database, and without a permanent database, hypertext
loses most of its utility.
